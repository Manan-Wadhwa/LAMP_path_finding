## 1. Why MaxEnt IRL Cannot Scale Here

This section is a hard break from task1. Understanding why matters before choosing replacements.

### 1.1 What MaxEnt IRL actually requires

Ziebart et al. (2008) MaxEnt IRL recovers a linear reward function r(s) = theta^T phi(s) such
that the probability of a trajectory tau = (s0, a0, s1, ...) is:

    P(tau | theta) proportional_to exp( sum_t r(s_t) ) = exp( theta^T sum_t phi(s_t) )

The parameter vector theta is found by maximizing the likelihood of expert demonstrations
D = {tau_1, ..., tau_n}. The gradient is:

    nabla_theta L = sum_{tau in D} phi(tau)/|D|  -  E_{tau~P(tau|theta)}[phi(tau)]

The second term requires computing the expected state-visitation frequency under the current
policy via a forward-backward pass on the MDP. This term vanishes — theta is completely
unidentified — when |D| = 0.

### 1.2 Why the task1 output cannot serve as demonstrations

**Problem A — Circular inference.**
The task1 route is the output of an IRL run over three points and a terrain cost surface. Using
it as a demonstration for a second IRL run means: learning weights that reconstruct the path
that was itself generated by assumed weights. The second IRL converges on weights that perfectly
reproduce the first path, regardless of what the true historical paths were. You learn nothing.

**Problem B — Underdetermination at scale.**
Three demonstration points generate at most 3 pairwise paths = 3 independent route segments.
The feature vector phi over a raster with k features has k free parameters. With k >> 3
(terrain, spectral bands, spectral indices, distance-to-buildings — easily 10-20 features), the
IRL is underdetermined by more than an order of magnitude even if the demonstrations were valid.

**Problem C — No analogous annotated site.**
Cross-site transfer learning requires a necropolis of similar layout, density, and terrain type
where historical paths have been excavated or reliably documented. No such dataset exists for a
desert Roman-period necropolis of this morphology. Pompeii's street network is excavated but its
geometry, density, and terrain context are entirely different. Amarna is a flat city plan, not a
hillside necropolis. Transfer is not justified.

### 1.3 The replacement strategy: 5-stream evidence ensemble

Replace IRL with a 5-stream evidence ensemble, each stream producing its own independent
path-hypothesis raster, then combine:

| Stream | Method family | Needs demonstrations? | Evidence type |
|---|---|---|---|
| A | Terrain-informed FETE | No | Indirect (movement mechanics) |
| B | Electrical circuit model | No | Indirect (random walk theory) |
| C | WorldView-2 multi-temporal spectral analysis | No | DIRECT (physical surface evidence) |
| D | Space syntax / axial analysis | No | Indirect (perceptual/social logic) |
| E | Proximity graphs (Gabriel, beta-skeleton) | No | Structural (geometric necessity) |

These are described mathematically below and implemented in Phases 4-10.

## 2. Mathematical Foundations

### 2.1 Anisotropic Cost Surface

Movement cost through a raster cell is direction-dependent (traversing a slope costs less going
downhill than uphill). This requires an anisotropic formulation.

**Tobler's hiking function (Tobler 1993):**

    W(s) = 6 * exp(-3.5 * |s + 0.05|)    [km/h]

where s = delta_z / delta_d is slope (signed: positive = uphill, negative = downhill).

Asymmetric travel cost per metre:

    C_tobler(s, d) = d / W(s)    [seconds]

**Llobera-Sluckin energy cost (Llobera & Sluckin 2007), derived from biomechanical principles:**

    E(theta) = a * (1 + b*sin^2(theta) + c*theta^2)    [J kg^-1 m^-1]

where theta = arctan(s), a = 1.5, b = 5.9, c = 0.17. Shown to outperform Tobler in
reconstructing real Roman roads (Verhagen 2013). Implement both; compare outputs.

**Multi-band composite cost function:**

    C(i,j) = w1*C_terrain(i,j) + w2*C_obs(i,j) + w3*C_spec(i,j) + w4*C_sar(i,j)

where:
- C_terrain = Tobler or Llobera-Sluckin cost derived from DEM slope
- C_obs = infinity inside building footprint mask (hard no-go), 0 elsewhere
- C_spec = derived from WV-2 spectral path indicator — low where spectral path evidence is high
- C_sar = derived from backscatter / SAR-derived products; low in smooth linear zones

Weights w1-w4 are named parameters, not magic numbers. Document them. Run sensitivity analysis
over a grid of weight combinations to validate robustness.

**48-neighbourhood kernel (Harris 2000):** For a move from (i,j) to neighbour (i+di, j+dj),
the diagonal correction factor is sqrt(di^2 + dj^2), keeping path-length distortion below 1.4%
versus the true optimum (compared to ~8% for an 8-connected grid). skimage.graph.MCP_Geometric
handles this automatically with fully_connected=True.

### 2.2 FETE and Movement Potential (Stream A)

Let E = {e_1, ..., e_n} be the set of n entrance points (n <= 342).

**Least-cost path:** For source e_k, the accumulated cost surface ACC_k(i,j) is computed by
multi-source Dijkstra from e_k over C(i,j). The LCP from e_k to e_l is the traceback through
ACC_k from e_l.

**Movement potential (Llobera 2000):**

    MP(i,j) = sum_k sum_{l != k}  indicator[ (i,j) in LCP(e_k, e_l) ]

Every pair of entrances contributes 1 to each cell it passes through. High-MP cells are
"where corridors converge" and are the best candidates for real paths.

**Computational complexity:** For n entrance nodes over a raster of V cells:
- One multi-source Dijkstra: O(V log V) with a binary heap
- For n entrances: n runs = O(n * V log V)
- For n=342, V = ~1,000,000 (1000x1000 grid): ~342 * 10^6 * 20 ops = ~7 * 10^9 ops
- With skimage.graph.MCP_Geometric (C-backed): under 5 minutes on a modern machine

CRITICAL: Do not run pairwise search n(n-1)/2 times. Run n Dijkstra passes (one per source),
then extract all tracebacks from each accumulated surface. This is the FETE architecture.

### 2.3 Electrical Circuit Model (Stream B)

Random walk theory establishes a formal equivalence between electrical networks and stochastic
movement (Doyle & Snell 1984; McRae et al. 2008):

**Resistance network construction:**
- Each raster cell (i,j) has conductance G(i,j) = 1 / C(i,j)
- Edge between adjacent cells: conductance = harmonic mean of cell conductances

**Kirchhoff equations:** For source node s injecting current I=1 and sink node t, the voltage
vector V satisfies:

    K * V = b

where K is the sparse conductance matrix (graph Laplacian) and b is the current injection
vector (b[s]=1, b[t]=-1, 0 elsewhere).

**Current flow density:**

    I(i,j) = G(i,j) * |V(i,j) - V(neighbor)|

Accumulated over all source-sink pairs, I(i,j) gives a probability-theoretic movement density.

**Key advantages over FETE:**
1. Flows through ALL paths simultaneously, weighted by conductance. FETE picks only the single
   least-cost path per pair; the circuit model distributes current proportionally through all
   paths. This naturally represents uncertainty about which specific route was used.
2. Resistance distance is formally the commute time of a random walk (McRae et al. 2008,
   Theorem 1). This grounds path probability in proven Markov chain theory, not heuristics.
3. No threshold decisions at inference stage — current is continuous.

**Solvers:** scipy.sparse.linalg.spsolve (direct, for small grids) or LGMRES with ILU
preconditioner (iterative, for large grids). For V=10^6 cells, the conductance matrix K has
~8*10^6 non-zero entries; LGMRES converges in seconds.

**Super-node formulation (efficient multi-pair):** Connect all entrance nodes to a meta-source
and all non-entrance nodes to a meta-sink with small conductance; solve once. This gives the
joint movement potential in O(1) linear system solve rather than O(n^2) solves.
See McRae et al. (2008, section 3.3).

### 2.4 WorldView-2 Spectral Path Indicators (Stream C)

**Physics of desert compacted paths:** Foot traffic over centuries consolidates loose surface
sediment, removing fine-fraction material, reducing micro-roughness, and slightly altering soil
mineralogy through iron oxidation and carbonate exposure. In arid environments (low biological
activity), this signal is often persistent over centuries.

**Observable signatures in WV-2:**

(a) Iron Oxide Ratio (IOR):
    IOR = Band5 / Band3
Disturbed/compacted soils often show elevated IOR due to ferric iron exposure. Ancient paths
in arid zones have been detected via IOR in multiple remote sensing studies.

(b) Soil-Adjusted Vegetation Index (SAVI):
    SAVI = (Band7 - Band5) / (Band7 + Band5 + 0.5) * 1.5
Paths have even lower SAVI than surrounding desert — no micro-level lichen/moss. Low SAVI in
linear patterns = candidate path evidence.

(c) NDRE (Red Edge):
    NDRE = (Band7 - Band6) / (Band7 + Band6)

(d) Surface Brightness (ALB):
    ALB = (Band2 + Band3 + Band4 + Band5) / 4
Compacted surfaces may have slightly elevated albedo at low sun angle.

**Multi-temporal stability index (the most powerful indicator):**

Across the three acquisition dates (March 22, June 1, June 14, 2018), most desert surfaces show
subtle seasonal variation. Compacted paths are **temporally stable** — same spectral signature
regardless of season.

For each pixel (i,j) and band b:
    mu_b(i,j) = mean( B_b_P001(i,j), B_b_P002(i,j), B_b_P003(i,j) )
    sigma_b(i,j) = std( B_b_P001(i,j), B_b_P002(i,j), B_b_P003(i,j) )
    CV_b(i,j) = sigma_b(i,j) / mu_b(i,j)   [coefficient of variation]

Low CV composite (Spectral Path Indicator):
    SPI(i,j) = 1 - mean_b( CV_b(i,j) )

High SPI = temporally stable = candidate path zone. Entirely independent of all modelled cost
surfaces — the ONLY direct physical evidence channel in this pipeline.

**Frangi vesselness filter (Frangi et al. 1998):** Applied to the pan-sharpened image (0.46 m
GSD) to detect thin linear features. At 0.46 m GSD, a 2-metre-wide path spans ~4 pixels — at
the detection limit of the Frangi filter.

Compute 2D Hessian of smoothed image at scale sigma:
    H_sigma = I * G_sigma''(x,y)   [convolution with second-derivative-of-Gaussian]

Eigenvalues lambda_1 <= lambda_2 of H_sigma. For bright line on dark background: lambda_1 = 0,
lambda_2 << 0.

Vesselness response:
    V_0(sigma) = 0                                    if lambda_2 > 0
               = exp(-R_B^2/2*beta^2) * (1-exp(-S^2/2c^2))  otherwise
    R_B = |lambda_1/lambda_2|,   S = sqrt(lambda_1^2 + lambda_2^2)

Final scale-space maximum: V(x,y) = max_sigma V_0(sigma) over sigma in {1, 2, 4, 8} pixels.
Also apply to dark-on-bright variant (paths may be darker due to compaction removing bright
top-layer dust) and take maximum of both.

### 2.5 Space Syntax (Stream D)

Space syntax (Hillier & Hanson 1984) quantifies how well-connected each space is to all others
— capturing the **social and perceptual logic** of movement that terrain-cost models miss. In a
settlement-like necropolis, routes that maximise accessibility (not just minimise effort) are
plausible primary-street candidates.

**Axial lines:** The longest straight lines covering all convex spaces in the free space
(complement of building footprint union). Generate algorithmically:
1. Compute binary free-space mask: erode BuildingFootprints union by ~0.5 m buffer
2. Extract medial axis / skeleton of free space
3. For each skeleton branch, fit the longest possible line contained within free space
4. Prune: remove lines whose angular deviation from skeleton axis exceeds 15 degrees

**Integration (global accessibility measure):**
    MD(i) = sum_j d(i,j) / (n-1)          [mean topological depth]
    RA(i) = 2*(MD(i)-1) / (n-2)           [relative asymmetry]
    RRA(i) = RA(i) / D_n                   [normalized — diamond baseline]
    Integration(i) = 1 / RRA(i)

High Integration = most accessible = candidate primary route. Penn et al. (1998) demonstrated
that integration predicts pedestrian movement rates in urban environments with R^2 = 0.5-0.8.

**Implementation:** momepy library provides space syntax metrics on networkx graphs.

### 2.6 Proximity Graphs (Stream E)

Proximity graphs provide a **geometry-first baseline**: given only entrance point locations
(no terrain, no imagery), what edges are structurally necessary to connect them?

**Gabriel Graph (GG):** Edge (u,v) exists if and only if the diametric circle on uv contains
no other point:
    (u,v) in GG   iff   forall w != u,v:  d(u,w)^2 + d(v,w)^2 > d(u,v)^2

The Gabriel graph is a subset of the Delaunay triangulation.

**Beta-skeleton:** A one-parameter family generalizing GG (beta=1) toward RNG (beta=2).
For archaeological settlement networks, beta = 1.0-1.5 typically gives the right connectivity.
Run a sweep over beta in {1.0, 1.1, 1.2, 1.3, 1.4, 1.5, 2.0} and overlay with the FETE
density map to find which beta aligns best with high-density corridors.

**Steiner Tree:** Given terminal nodes T (the entrance set), the Steiner tree is the
minimum-cost tree spanning all terminals, allowing Steiner points anywhere in the graph.
- networkx provides steiner_tree (Kou et al. approximation, ratio <= 2)
- Real streets are more than the minimum (redundant routes, loops, dead-end spurs), but the
  Steiner tree identifies which connections are structurally unavoidable — useful lower bound.

### 2.7 Ensemble Confidence Scoring

Five evidence streams produce five rasters, each normalized to [0,1]:

| Stream | Raster | Interpretation |
|---|---|---|
| A | fete_density_norm | High = many LCPs converge here |
| B | circuit_current_norm | High = strong random-walk flow |
| C | spi | High = temporally stable surface (DIRECT EVIDENCE) |
| D | space_syntax_integration_norm | High = topologically central |
| E | proximity_edge_density | High = many proximity graph edges pass through |

**Weighted ensemble:**
    confidence(i,j) = w_A*fete + w_B*circuit + w_C*spi + w_D*syntax + w_E*prox

Default weights (to tune):
    w_A=0.30, w_B=0.25, w_C=0.25, w_D=0.10, w_E=0.10

Rationale: FETE and Circuit are both terrain-based but methodologically independent. SPI is the
only direct physical evidence stream; upweight relative to purely model-based streams. Proximity
is a geometric constraint; least sensitive to terrain.

**Per-segment confidence fields in final GeoJSON:**
- confidence_fete, confidence_circuit, confidence_spi, confidence_syntax, confidence_ensemble
- method_agreement: count of streams with score > 0.5 (1-5)
- flag_review: True if method_agreement <= 2
